What is the Travelling Salesman Problem (TSP)?

The Travelling Salesman Problem (TSP) is a classic algorithmic problem in the fields of computer science and operations research. It focuses on finding the shortest possible route that visits each city once and returns to the origin city. TSP is NP-hard, meaning it is computationally difficult as the number of cities increases. It has applications in logistics, transportation, and circuit design.




Algorithms Used

1. Nearest Neighbour Algorithm: A greedy algorithm that selects the nearest unvisited city at each step. It provides a fast but approximate solution.

2. Held-Karp Algorithm: A dynamic programming solution to TSP with improved accuracy. It uses memoization to avoid redundant computations and guarantees the optimal solution for small to medium-sized inputs.
